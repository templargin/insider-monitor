# insider-monitor

Daily small-cap insider purchase screener. Form 4 filings filtered to:

- Transaction code **P** (open-market purchase) only
- **One reporter (same CIK) â‰¥ $100k** in the bucket â€” two different insiders at $60k each do *not* qualify
- Enterprise value **< $1B** (basic shares)
- TTM revenue **> 0** â€” only revenue facts whose `end` date is within ~18 months of today count (otherwise a shell-co predecessor's ancient revenue, e.g. KLRS's 2019 $165k, leaks through after a reverse merger). Bank-style top-line tags (`InterestAndDividendIncomeOperating`, `NoninterestIncome`) are included in the screener's revenue ladder so community banks like BCML/CUBI/FMBM that don't tag `Revenues` aren't false-rejected.

Static site published to GitHub Pages at <https://templargin.github.io/insider-monitor>.

## URLs

- `/insiders/YYYY/` â€” months
- `/insiders/YYYY/month/` â€” weekdays
- `/insiders/YYYY/month/DD/` â€” daily list (URL date = the day you read; content = previous weekday's filings; Monday rolls in Fri+Sat+Sun). A bucket whose dates are *all* weekend/holiday (e.g. the Monday after Juneteenth) renders an explicit empty page (HTTP 200), not a 404 â€” see *Page-write safety* below.
- `/companies/TICKER/` â€” canonical per-ticker page with insider activity, valuation, and financial statements

## How it runs

Primary trigger is a cron on the `aspancrm-claude` droplet at `30 6 * * 1-5` ET (6:30am ET). It pulls main, dispatches GH Actions `daily.yml`, waits for completion, runs the LLM options/warrants extraction, then rebuilds + pushes. GH Actions also has its own `30 10 * * 1-5` UTC schedule as a fallback. See [CRON_DROPLET.md](CRON_DROPLET.md).

## Page-write safety

Each weekday's page is generated by both the droplet run and the GH Actions fallback (~3â€“4 h apart), so the daily writer (`scraper/pipeline.py::process_bucket`) is built to never publish a page it can't stand behind:

- **Transient data outages never blank or clobber a page.** `screener_pass` distinguishes *"couldn't fetch the inputs"* (SEC `companyfacts` or the yfinance share price) from *"screened out on EV/revenue merits"* â€” the former raises `DataUnavailable`, the latter returns `None`. If a run can't evaluate a **single** candidate issuer (a mass throttle of SEC or the price source â€” common on the delayed fallback run from cloud IPs), it **skips the write** instead of overwriting a good page with an empty one. It also never downgrades an existing non-empty page to empty on any run where a fetch errored. SEC fetches (`edgar._get`) and the price fetch retry with exponential backoff. *(This is the fix for the empty 2026-06-19 page: a fallback run hit a mass price-fetch failure, all 14 candidates returned "data unavailable", and an empty list overwrote PRTA + GOTU.)*
- **Genuinely empty buckets still publish.** When every filing-date in a bucket is a non-trading day (weekend or US federal holiday, per `buckets.is_trading_day` / `us_federal_holidays`), there will never be filings, so the page is written **empty (HTTP 200)** rather than 404ing. Good Friday is intentionally treated as a trading day â€” NYSE closes but EDGAR still accepts filings.
- **A real trading day that returns 0 index rows is treated as "not published yet"** (a too-early evening/early run) and skipped, so it can't clobber a good page either.

## Capital structure & debt

The cap-structure **Total debt** (and therefore EV) comes from `scraper/xbrl_statement.py::get_structured_debt`, which classifies borrowings by concept *family* and the us-gaap debt *hierarchy* rather than an exact allow-list of leaf tags. Three properties make it correct where the old leaf-ladder silently failed:

1. **Date-anchored** (`xbrl_facts.balance_sheet_date` + `instant_value_at`). Every point-in-time read is pinned to the latest balance-sheet date, so a tag a filer abandoned years ago can't leak in. This alone fixed ~29 stale figures â€” e.g. AVD read **$44M off a 2016 `LongTermDebt` fact** while its real **$266M** sat in the current `LongTermDebtNoncurrent` tag (the stale aggregate short-circuited the live component); ANIK showed an **$8M ghost from 2013** against a now debt-free sheet; LEE showed **$506M (2020)** vs **$455M** today.
2. **Classified by the us-gaap debt hierarchy, not a tag list.** Borrowings are recognized by family pattern (`LineOfCredit`, `SeniorNotes`, `ConvertibleNotesPayable`, `LongTermLineOfCredit`, finance leases, â€¦) and deduped by *tier* â€” a reported subtotal (`LongTermDebtNoncurrent`) subsumes the instruments beneath it, so AVD's `LongTermDebtNoncurrent` **$264M** + `LineOfCredit` **$283M** doesn't double to $547M. Beyond the fixed tiers, `_drop_family_subtotals` removes any tag whose local name is contained in â‰¥2 other matched tags summing (within 2%) to its value â€” so ANGX's `NotesPayable` **$102M** + its parts `LongTermNotesPayable` **$62M** + `NotesPayableCurrent` **$40M** is taken as $102M, not $205M. When a filer reports **no** balance-sheet debt line at all, the extractor falls back to the debt-footnote total `DebtInstrumentCarryingAmount` (EPSN's **$45.5M**, flagged `debt_from_footnote_total`). This captured debt the ladder reported as **$0**: REI's **$426M** revolver, VTS's **$144M**, AFCG's **$76M** senior notes, XRN's **$663M** (whose true EV crosses the $1B screen). Asset look-alikes are excluded â€” notes/loans *receivable* (DAVE/BNPL loan books), AFS debt *securities*, FHLB *stock*, `*Gross` duplicates.
3. **Flag, never plug** (move 3). The result is reconciled against the filer's reported total liabilities; residual uncertainty is surfaced on the page (`valuation.debt_flag`, rendered as a âš  + warning) instead of being absorbed into an "Other" plug. Flag reasons: `debt_tags_overlap_clamped` (raw sum exceeded total liabilities â†’ overlapping tags double-counted; falls back to the largest single instrument that fits, never a plug of unclassified liabilities â€” OPAD, BZUN), `unexplained_liabilities` (a material slice of recognized-debt-bearing filers' liabilities is neither recognized debt nor a named non-debt item), `debt_from_footnote_total` (debt taken from the footnote carrying amount because no balance-sheet debt line is tagged â€” EPSN), `financial_institution` (deposits are a large share of the balance sheet â†’ bank/thrift, EV isn't a meaningful metric).
4. **Audited on both sides.** Verifying captured debt is not enough â€” the silent EPSN miss and the ANGX over-count both slipped a single-sided check. Two committed scripts audit each direction against independent SEC ground truth: `scripts.audit_debt_free` scans every **debt-free** company for a borrowing the extractor missed; `scripts.audit_debt_value` cross-checks every **reported figure** against the filer's own `DebtInstrumentCarryingAmount` / `LongTermDebt` totals. Both report **0** across all 196 stored companies. Note the reach limits: the value audit only covers filers that report a reference total, and debt under a filer's *custom namespace* is invisible to companyfacts and so to both audits.

The test of the design: there is **no list of debt leaf tags to extend**. A filer's next exotic borrowing tag is captured by family pattern + tier dedup, or it surfaces as a flag â€” never as a silent $0.

## Financial statements

Per-ticker pages pull IS / BS / CF directly from SEC XBRL `companyfacts`. Rows are designed to reconcile by construction:

- **LTM column** is the leftmost period on annual IS / BS / CF / Ratios. Flow items (Revenue, expenses, NI, EPS) sum the last 4 quarterly values; point-in-time items (Diluted/Basic Avg Shares) take the latest quarterly value; BS uses the most-recent quarter-end values. Margins are derived from the column directly so LTM Margin = LTM_NI / LTM_Revenue reconciles.
- **Cash-flow operating section foots by construction.** The XBRL extractor pulls only Net Income, D&A, and Stock-Based Comp explicitly, so a single derived row **Î” Working Cap & Other** = OCF âˆ’ (NI + D&A + SBC) absorbs the change in working capital plus every other non-cash adjustment the filer discloses. Without it the operating section never reconciled from Net Income to Operating Cash Flow (gaps frequently exceeded OCF itself). `python -m scripts.audit_cashflow` now checks this bridge in addition to the bottom-line Î”Cash identity.
- **Every statement foots from its line items by construction.** A derived plug row reconciles each subtotal to the items above it: the income statement's Operating Income â†’ Pretax bridge (`Other Income / (Expense)` = Pretax âˆ’ OpInc + Interest, sign-robust); the cash-flow investing section (`Other Investing` = Investing CF âˆ’ CapEx âˆ’ Acquisitions) and financing section (`Other Financing` = Financing CF âˆ’ debt/stock/dividend lines); alongside the operating section and balance sheet below. Audited with `/tmp/audit_v2.py`-style checks that test each subtotal against its visible items in *both* directions.
- **Balance sheet foots from line items by construction.** Each subtotal is reconciled to its components via a derived plug row â€” `Other Current Assets` = Total Current Assets âˆ’ (Cash + ST Investments + Receivables + Inventory); `Other Assets`, `Other Current Liabilities`, `Other Liabilities` analogously (unclassified bank/REIT sheets roll everything between the named items and the total into the `Other` plug). The line-item set was expanded to capture what was previously buried in the totals: Short-term Investments, Operating Lease ROU assets, Intangibles, Long-term Investments on the asset side; Current Debt, Current/Long-term Lease Liabilities on the liability side. The Net PPE ladder leads with oil & gas property (VTS) and PP&E-incl-finance-lease (OPAL) so an energy filer's real asset base isn't lost. A line item that exceeds its enclosing subtotal (a mistagged debt-maturity schedule or a non-current security tagged "current" â€” STRZ/LOGC/GPUS) is dropped into the plug rather than producing a negative `Other` row.
- **Weighted-average share counts** use only directly-reported discrete quarters â€” the YTD-subtraction walk (FY âˆ’ 9M = Q4) is disabled for them because share counts are not additive (it previously produced absurd values such as BKKT's âˆ’6.3M "Q4 diluted shares"). A separate EPS-consistency pass corrects clean order-of-magnitude unit errors (FMBM tagged 3,560M vs 3.55M actual) using the filer's own EPS as ground truth, while deliberately leaving reverse-split / minority-interest artifacts untouched.
- **Operating Expense** is derived as Gross Profit âˆ’ Operating Income (avoids the inconsistent `OperatingExpenses` XBRL tag that double-counts COGS for many filers).
- **Operating Income** itself falls back to `Revenue âˆ’ CostsAndExpenses` when the `OperatingIncomeLoss` tag is missing (filers like BH, STRZ that don't tag OpInc directly).
- **Pretax Income** is derived as Net Income + Tax (avoids the `*MinorityInterest*` Pretax variant that some filers sign-flip for losses).
- **Net Income** = `ProfitLoss` (includes NCI); a separate **Net Income to Common** row appears when NCI is meaningful (e.g., MKTW) so EPS Ã— shares â‰ˆ parent's NI is visibly consistent.
- **Total Equity** uses the including-NCI variant; a **Mezzanine Equity** row appears for redeemable preferred (e.g., BETR FY22) so Assets = Liabilities + Mezzanine + Total Equity holds.
- **Total Revenue** for banks/insurance uses `InterestAndDividendIncomeOperating + NoninterestIncome` as primary (otherwise BWFG-type filers would show just contract-customer fee income, wildly understating revenue).
- **Cost of Revenue** ladder covers the long tail of filer-specific tags: `CostOfGoodsAndServiceExcludingDepreciationDepletionAndAmortization` (services, e.g., DLHC), `DirectOperatingCosts` (media/content licensing, e.g., STRZ), `CostOfRealEstateSales` (real estate, e.g., AXR), and a `CostDirectMaterial + CostDirectLabor` sum for restaurants (PTLO).
- **Gross Profit / Gross Margin % are mandatory and never blank.** A per-period derivation ladder fills GP for every filer, most-specific archetype first, then a universal floor:
  1. Reported `GrossProfit` tag, else Revenue âˆ’ Cost of Revenue (CoR ladder below).
  2. **Services fallback** GP = OpInc + G&A (filers that report Operating Income and G&A but no CoR â€” PFHO, AXR, TOI, etc.). Gated *per period*: skipped when R&D â‰¥ 50% of revenue (genuine biotechs) or when the bank tag `NoninterestExpense` is present, and accepted only when the implied GP lands in [0, 1.05Â·Rev] so margin stays in a sane 0â€“100% band. The old gate was whole-statement `ResearchAndDevelopmentExpense in usg`, which wrongly blanked operators that merely tag a token R&D line (FONR â€” medical imaging, R&D = 1.5% of revenue, now correctly ~40%).
  3. **Bank Net Interest Income** GP = interest income âˆ’ interest expense (`InterestAndDividendIncomeOperating` / `InterestIncomeOperating` / `InterestAndFeeIncomeLoansAndLeases`, minus `InterestExpense`). Excludes `InterestIncomeExpenseNet` so non-banks that carry a small net-interest line aren't mistaken for banks. Covers BCML, BWFG, CARE, CUBI, FMBM, NTB, PNBK, UBCP, AFCG, PFBX.
  4. **Insurance underwriting margin** GP = `PremiumsEarnedNet` âˆ’ `PolicyholderBenefitsAndClaimsIncurredNet` (AII, KINS, UTGN).
  5. **Universal revenue floor** GP = Revenue (implied CoR = 0) for anything still missing â€” correct for pre-revenue clinical biotechs and early SaaS (ABSI, ZBIO, AUID, â€¦), and the backstop that guarantees the row is never empty. A handful of large-revenue filers with non-standard cost tags (BKKT crypto, LEE media, PROP oil & gas annual) land here and show 100% â€” a known overstatement pending archetype-specific CoR synthesis.
- **Cost of Revenue reconciles to the gross block**: wherever a CoR value is shown it is rewritten to Rev âˆ’ GP, so Rev âˆ’ CoR = GP always holds on screen. Fixes filers (SLNH, GVH, MIMI) whose partial CoR tag disagreed with their authoritative reported GrossProfit subtotal. Banks/insurers/floor filers that report no CoR keep an empty row (no invented number).
- The bank interest-income tags are also in the **Total Revenue** ladder (incl. bare interest income as a last resort) so thrifts like PFBX that tag neither a standard revenue line nor `NoninterestIncome` still show a top line and therefore a margin.
- **Quarterly cadence guards**: the YTD walk (FY âˆ’ 9M = Q4, etc.) dedupes restated XBRL facts by `(end, duration-band)` first and only emits a derived quarter when the implied subtraction span lands in the Q band (60â€“100d). Annual-only filers (SMWB, 20-F foreign) and semi-annual filers (UCAR, GVH â€” H1 + FY only) therefore collapse to "No quarterly data available" instead of fake year-end zeros or H2 totals mislabeled as Q4.
- **Quarterly YoY** compares each quarter to the same quarter one year prior (8-quarter internal buffer, 4 displayed).
- **Effect of FX on Cash** row in CF for foreign-operations filers.

Run `python -m scripts.audit_financials` for an end-to-end reconciliation pass (GP, OpInc, NI, A=L+M+E, share sanity). `python -m scripts.audit_cashflow` for CF reconciliation (Î”Cash vs OCF+ICF+FCF+FX). `python -m scripts.probe_coverage` reports per-row coverage % across all stored tickers and ranks candidate XBRL tags that would unblock the most missing cells â€” re-run when a ticker's page looks empty to identify what tag to add to a ladder.
